## 🔹 MODULE 1 — Comprendre les KPI RH
**Niveau :** Débutant | **Durée :** 3h | **REAC CP3 :** Identifier les indicateurs RH pertinents

### Objectifs pédagogiques (SMART)
À l'issue de ce module, le stagiaire sera capable de :
1. Définir un KPI et le distinguer d'une donnée brute
2. Expliquer le rôle des KPI dans la stratégie RH
3. Classer les KPI par famille (recrutement, social, formation, paie)
4. Identifier 3 KPI pertinents pour son poste d'ARH

### 🎬 Storytelling d'entrée
> Sophie, assistante RH chez NeoCorp (350 salariés, secteur tech), reçoit un mail de la DRH un lundi matin : *"Sophie, la direction demande un point KPI RH pour jeudi. Prépare quelque chose."* Sophie panique. Elle a des données partout (Excel, SIRH, e-mails) mais ne sait pas quoi mesurer ni comment le présenter. Ce module est son point de départ.

### 📚 Contenu théorique (capsule 8 min)

**1. Qu'est-ce qu'un KPI ?**
- KPI = Key Performance Indicator = Indicateur Clé de Performance
- Différence donnée brute / indicateur / KPI :
  - Donnée : "47 absences en mars"
  - Indicateur : "Taux d'absentéisme = 4,3%"
  - KPI : "Taux d'absentéisme cible < 3,5% — ALERTE rouge"
- Un KPI est SMART : Spécifique, Mesurable, Atteignable, Réaliste, Temporel

**2. Pourquoi les KPI RH ?**
- Piloter la performance humaine de l'entreprise
- Anticiper les risques (turnover, absentéisme, sous-effectif)
- Justifier les décisions RH auprès de la direction
- Obligation légale (BDES, bilan social)

**3. Les 4 familles de KPI RH**
| Famille | Exemples |
|---------|----------|
| Recrutement | Délai de recrutement, coût par recrue, taux de rétention N+1 |
| Social | Taux d'absentéisme, taux de turnover, satisfaction salariés |
| Formation | Taux de réalisation du plan, heures de formation/salarié |
| Paie & RH Admin | Masse salariale/CA, coût moyen chargé, taux d'erreur paie |

**4. Les caractéristiques d'un bon KPI**
- Actionnable : il doit permettre de décider
- Comparé : à une cible, à N-1, au secteur
- Visualisé : couleur, graphique, tendance
- Actualisé : fréquence définie (mensuel, trimestriel)

### 🧠 Exercice interactif — Tri des KPI

- Nombre de CV reçus → Recrutement

- Heures CPF consommées → Formation
- Charges patronales → Paie
- Délai moyen de recrutement → Recrutement
- Taux de promotion interne → Social/RH

### 🎬 Storytelling d'entrée

### 📚 Contenu théorique (capsule 7 min)

**1. Les 3 niveaux de KPI**
| Niveau | Qui l'utilise | Exemples |
|--------|--------------|----------|
| Stratégique | Direction, DRH | Masse salariale/CA, turnover cadres |
| Opérationnel | ARH, Manager | Délai recrutement, taux absentéisme service |
| Réglementaire | RH, Juridique | Index égalité H/F, taux emploi handicapé |

**2. Comment choisir ses KPI ?**
- Partir des enjeux de l'entreprise (croissance ? fidélisation ? réduction coûts ?)
- Vérifier que la donnée est disponible et fiable
- Limiter à 5–8 KPI par tableau de bord (loi de Miller)
- Associer chaque KPI à un responsable et une fréquence

**3. KPI incontournables pour un ARH**
| KPI | Formule simplifiée | Fréquence |
|-----|--------------------|-----------|
| Taux d'absentéisme | Jours absents / (Eff × jours ouvrés) × 100 | Mensuel |
| Taux de turnover | Départs / Effectif moyen × 100 | Mensuel/An |
| Délai moyen de recrutement | Σ délais / Nb recrutements | Par vague |
| Taux de réalisation entretiens | Entretiens réalisés / Effectif × 100 | Annuel |
| Coût moyen de recrutement | Budget recrutement / Nb embauches | Annuel |

**4. Les pièges à éviter**
- Trop de KPI → paralysie par l'analyse
- KPI sans cible → impossible à interpréter
- KPI non actualisés → décisions sur données périmées
- KPI non partagés → silotage de l'information

## 🔹 MODULE 3 — Calcul des KPI
**Niveau :** Intermédiaire | **Durée :** 4h | **REAC CP3 :** Calculer et vérifier les indicateurs RH

### Objectifs pédagogiques (SMART)
1. Appliquer les formules des 8 KPI RH fondamentaux
2. Identifier et corriger les erreurs de calcul courantes
3. Calculer un taux d'absentéisme, de turnover, de coût de recrutement sur données réelles
4. Interpréter un résultat en le comparant à une référence sectorielle

### 🎬 Storytelling d'entrée
> Sophie a son référentiel de KPI. La DRH lui demande maintenant de calculer les chiffres pour le rapport mensuel. Elle ouvre son Excel et réalise qu'elle ne maîtrise pas toutes les formules. Le module 3 lui donne les outils pour ne plus jamais bloquer sur un calcul.

### 📚 Contenu théorique (capsule 10 min)

**FORMULES FONDAMENTALES**

**1. Taux d'absentéisme**
```
TA = (Nombre de jours d'absence / Nombre de jours théoriques travaillés) × 100
Jours théoriques = Effectif × Nombre de jours ouvrés de la période
Référence sectorielle : < 3,5% (France, moyenne tous secteurs)
```

**2. Taux de turnover**
```
TTO = (Nombre de départs / Effectif moyen) × 100
Effectif moyen = (Effectif début + Effectif fin) / 2
Référence : < 15% considéré acceptable en France
```

**3. Taux de turnover volontaire**
```
TTV = (Démissions / Effectif moyen) × 100
Important : distinguer départs volontaires / involontaires
```

**4. Délai moyen de recrutement**
```
DMR = Somme des délais (affichage offre → date d'embauche) / Nombre de recrutements
Référence : 30–45 jours en PME
```

**5. Coût moyen de recrutement**
```
CMR = (Coûts annonces + Coûts cabinet + Temps RH valorisé + Intégration) / Nb embauches
Référence : 3 000–8 000 € selon poste
```

**6. Taux de réalisation plan de formation**
```
TRF = (Heures de formation réalisées / Heures planifiées) × 100
Cible : > 85%
```

**7. Masse salariale / Chiffre d'affaires**
```
MS/CA = Masse salariale brute chargée / Chiffre d'affaires × 100
Référence : 20–40% selon secteur
```

**8. Taux de réalisation des entretiens annuels**
```
TEA = (Entretiens réalisés / Effectif éligible) × 100
Cible légale : 100% (obligation CPF/entretien pro tous les 2 ans)
```

**Erreurs fréquentes à éviter :**
- Confondre effectif physique et ETP
- Oublier les CDD dans le calcul du turnover
- Utiliser des jours calendaires au lieu des jours ouvrés
- Ne pas distinguer absences justifiées / injustifiées

### 🧠 Exercice interactif — Calculatrice KPI
*Complétez le tableau avec les bonnes formules et résultats :*

| Données NeoCorp — Avril | Valeur |
|-------------------------|--------|
| Effectif début | 350 |
| Effectif fin | 347 |
| Jours d'absence | 68 |
| Jours ouvrés | 21 |
| Départs (dont 4 démissions) | 6 |
| Recrutements finalisés | 3 |
| Délais recrutement (j) | 28, 35, 42 |

*Calculez : TA, TTO, TTV, DMR*

**Corrections :**
- Effectif moyen = (350+347)/2 = 348,5
- TA = 68/(348,5×21)×100 = **0,93%** ✅ (objectif < 3,5% → vert)
- TTO = 6/348,5×100 = **1,72%** mensuel → **20,6% annualisé** ⚠️ (> 15% → orange)
- TTV = 4/348,5×100 = **1,15%** mensuel → **13,8% annualisé** (démissions)
- DMR = (28+35+42)/3 = **35 jours** (objectif 30j → légèrement hors cible)

### 💼 Cas pratique NeoCorp — Module 3
> NeoCorp vous transmet les données RH du S1 (6 mois) :
> - Effectif moyen : 348
> - Total jours d'absence : 892 (sur 130 jours ouvrés)

> - Entretiens annuels réalisés : 289 / 348 éligibles

**Correction complète :**

2. TTO = 28/348×100 = **8,05%** sur 6 mois → **16,1% annualisé** 🔴 (> 15%)

## 🔹 MODULE 4 — Analyse RH & Décision
**Niveau :** Intermédiaire | **Durée :** 3h | **REAC CP3 :** Analyser les données et proposer des actions

### Objectifs pédagogiques (SMART)
1. Interpréter un KPI en contexte (tendance, comparaison, saisonnalité)
2. Identifier les causes racines d'un KPI dégradé
3. Formuler 3 recommandations RH argumentées à partir de données
4. Rédiger une note d'analyse RH d'une page pour la direction

### 🎬 Storytelling d'entrée
> La DRH de NeoCorp reçoit Sophie : *"Les chiffres, je les vois. Ce que je veux, c'est savoir pourquoi et quoi faire."* Sophie comprend que son rôle va au-delà du calcul : elle doit analyser, croiser, interpréter et proposer.

### 📚 Contenu théorique (capsule 8 min)

**1. De la donnée à la décision**
```
Donnée brute → Calcul → Indicateur → Analyse → Recommandation → Décision
```

**2. Les 4 niveaux d'analyse**
| Niveau | Question | Exemple |
|--------|----------|---------|
| Descriptif | Que s'est-il passé ? | TA = 2,3% en mars |
| Diagnostique | Pourquoi ? | Pic grippe + 1 service sous-effectif |
| Prédictif | Que va-t-il se passer ? | Risque pic en décembre |
| Prescriptif | Que faire ? | Plan prévention, renfort service X |

**3. Croiser les KPI**
- Turnover élevé + recrutement difficile = risque de sous-effectif chronique
- Absentéisme en hausse + baisse satisfaction = signal d'alerte climat social
- Formation sous-réalisée + turnover fort = départ de compétences non remplacées

**4. Méthode des 5 Pourquoi (RCA)**
> Problème : Turnover 22% chez NeoCorp
> 1. Pourquoi ? → Beaucoup de démissions (18 sur 6 mois)
> 2. Pourquoi ? → Principalement des <2 ans d'ancienneté
> 3. Pourquoi ? → Mauvaise intégration (onboarding insuffisant)
> 4. Pourquoi ? → Pas de parcours d'intégration formalisé
> 5. Pourquoi ? → Pas de budget ni de référent dédié

**5. Structure d'une note d'analyse RH**
1. Contexte et périmètre
2. Principaux indicateurs (3–5 KPI avec statut RAG)
3. Analyse des écarts
4. Causes identifiées
5. Recommandations (court/moyen terme)
6. Indicateurs de suivi

- **Mission** : rédigez 3 hypothèses causales et 2 actions correctives

**1. Qu'est-ce qu'un tableau de bord RH ?**

```

│  TABLEAU DE BORD RH — NEOCORP — [MOIS/ANNÉE]        │
│  Effectif total : XXX | Période : du XX au XX        │

├─────────────────────────────────────────────────────┤

└─────────────────────────────────────────────────────┘
```

**3. Règles de visualisation**
- RAG : Rouge (hors cible >10%), Orange (hors cible <10%), Vert (dans la cible)

- Trop d'informations → effet catalogue
- Pas de cibles → impossible à interpréter
- Mise à jour irrégulière → perte de confiance
- Design trop complexe → inutilisé en réunion

```excel
=COUNTA()        → Compter l'effectif

=SUMIF()         → Sommer par critère (masse salariale par service)

Tableau croisé dynamique → Analyser n'importe quelle donnée RH
```

**2. Formules KPI en Excel**
```excel
-- Taux d'absentéisme
=B2/(C2*D2)*100
-- B2=jours absence, C2=effectif, D2=jours ouvrés

-- RAG automatique avec mise en forme conditionnelle
=IF(E2>3.5,"ROUGE",IF(E2>2.5,"ORANGE","VERT"))

-- Turnover annualisé
=F2/G2*100*12
-- F2=départs mois, G2=effectif moyen
```

## Logique de progression (JavaScript)

```javascript
// Après soumission d'un QCM
async function handleModuleCompletion(userId, moduleId, scoreQCM, scoreCas) {
  const scoreTotal = scoreQCM + scoreCas;
  const scorePct = (scoreTotal / 15) * 100;

  let niveauAdaptatif;
  if (scorePct < 50) {
    niveauAdaptatif = 'renforcement';
    // Déverrouiller module de renforcement
    await unlockRenforcementModule(userId, moduleId);
  } else if (scorePct < 80) {
    niveauAdaptatif = 'standard';
  } else {
    niveauAdaptatif = 'avance';
    // Déverrouiller cas avancé
    await unlockAdvancedCase(userId, moduleId);
  }

  // Sauvegarder progression
  await supabase.from('tb_kpi_progression').upsert({
    user_id: userId,
    module_id: moduleId,
    statut: 'termine',
    score_total: scoreTotal,
    niveau_adaptatif: niveauAdaptatif,
    completed_at: new Date().toISOString()
  });

  // Vérifier et attribuer badges
  await checkAndAwardBadges(userId);

  // Débloquer module suivant
  await unlockNextModule(userId, moduleId);
}

// Attribution automatique de badges
async function checkAndAwardBadges(userId) {
  const progression = await getUserProgression(userId);

  const badgeRules = [
    { code: 'ANALYSTE_KPI', condition: () => modulesValidated(progression, ['M1','M2'], 70) },
    { code: 'CALCULATEUR_RH', condition: () => moduleValidated(progression, 'M3', 70) },
    { code: 'PILOTE_DONNEES', condition: () => modulesValidated(progression, ['M4','M5'], 70) },
    { code: 'EXPERT_EXCEL', condition: () => moduleValidated(progression, 'M6', 80) },
    { code: 'SPECIALISTE_METIER', condition: () => moduleValidated(progression, 'M7', 80) },
  ];

  for (const rule of badgeRules) {
    if (rule.condition() && !badgeAlreadyAwarded(userId, rule.code)) {
      await awardBadge(userId, rule.code);
    }
  }
}
```
